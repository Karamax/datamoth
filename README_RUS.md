### Что такое datamot?

Datamot упрощает создание конвейерных пакетов обработки пакетов и решает большинство
которые возникают при развертывании рабочих мест в Hadoop / Oozie.

### Хорошо ... Так что это?

Datamot помогает вам организовать ваш аналитический код в проект и развернуть
это к клану хауопа. Вам не нужно копировать файлы в HDFS и заменять
переменные в конфигурации. Нет скриптов. Просто нажмите свой код на git
как вы делаете это всегда, и пусть волшебство произойдет.

### Хм .. что-нибудь еще?

Автоматический запуск / остановка заданий. Простая, централизованная и локальная конфигурация
проект. Конфигурационные профили (производство, испытание, что угодно). Свобода в проекте
состав.

### Это маркетинговый инструмент? Вы говорите, как продавец.

Полностью нет. Мы далеки от маркетинга, мы инженеры. Просто установите первый
версия datamot, и вы получите 50% скидку на следующую версию.

### Какие? Разве это не бесплатный инструмент?

Это бесплатно, и это бесплатно для коммерческого использования.

### Ну .. Хорошо, как его установить?

Datamot состоит из двух частей: cli tool (который вы можете использовать, но полностью не
должны) и супер простой веб-интерфейс. Когда вы запускаете веб-интерфейс в первый раз
он создаст голые репозитории git, каждый из которых будет указан в конфигурации.

Затем вы просто подталкиваете свой код к этим репозиториям и развертываете его в кластере.
Вся конфигурация кластера хранится в специальном файле conf в вашем проекте.
После нажатия вы можете перемещаться по своему проекту (наборы данных, координаторы, рабочие процессы,
кода) через веб-интерфейс, что очень помогает при устранении ошибок кода.

### Стоп! Пожалуйста остановись. Просто скажите, как его установить.

Загрузите дистрибутив. Распакуйте его. Запустите `datamot / bin / datamot --start`. Идти к
HTTP: // локальный: 2718.

### Так-то лучше. Теперь, как установить его, правильно?

Обычно вам необходимо установить datamot на какой-либо сервер, который доступен для push
через ssh для вас и ваших товарищей по команде. Ваш кластер hadoop и его службы, например
Oozie, HDFS должен быть доступен с этого сервера. И было бы просто приятно
создайте выделенного пользователя на этом сервере для запуска datamot.

После создания пользователя (или принятия решения об использовании существующего) распакуйте datamot
в любое место, которое вы хотите, и добавьте в папку `bin` путь. Итак, если для
Например, вы распаковываете его в `/ home / datamot / datamot`, тогда вам нужно исправить $ PATH
переменная окружения:

`$ PATH = / дом / datamot / datamot / bin /: $ PATH`

Теперь `datamot --version` должен работать где угодно. Если вы собираетесь использовать пользовательский интерфейс
(мы поклоняемся вам использовать его) просто запустите `datamot --start`. В настоящее время мы работаем
при установке datamot в качестве сервиса. Пока вы можете использовать экран для запуска отсоединенного
datamot вот так:

`screen -dmS datamot datamot --start`

Вы можете изменить ip по умолчанию и порт, на который будет прослушиваться дататат. Взгляни на
`Конф / datamot.conf`.

`remoteUser` является пользователем ssh, под которым вы собираетесь нажать git,
`remoteHost` - это хост для push,

эти две переменные необходимы для отображения URL-адреса push в пользовательском интерфейсе.

### Готово ли это производство?

Мы используем его на производстве, но на кластере с не так много (но довольно тяжелым)
рабочие места: около 30 координаторов. Сейчас мы находимся в раннем альфа-состоянии и
собрать больше отзывов от пользователей. Ждем ваших комментариев.

### Подождите ... Что такое координатор?

Хорошо, есть несколько менеджеров рабочих процессов для Hadoop. Один из лучших (нашими
мнение) - Оози. Вы знаете, это как контейнер для вас скриптов (питон, свинья,
hive-sql, что угодно), который периодически запускается планировщиком.

### Почему Оози?

Он декларативный и идеально подходит для современного взгляда на неизменную архитектуру.
У этого есть все необходимые сущности, которые помогут вам моделировать ваши расчеты: наборы данных,
рабочие процессы, координаторы, узлы-координаторы (конвейеры) и зависимости
между ними.

### Итак, что такое координатор?

Вам лучше посмотреть официальные документы Оози. Это отличный инструмент! Ну вот:
http://oozie.apache.org/.

### Получил, как запустить что-то на кластере, в конце концов?

Мы подготовим игрушечный проект для Hadoop / Oozie / Spark / Hive stack с несколькими
координаторов и простых вычислений игрушек в них в ближайшее время. Просто клонируйте его и
нажмите на URL, показанный в меню пользовательского интерфейса дататота. Но перед
нажмите файл fix .conf в корневом каталоге в соответствии с конфигурацией вашего кластера.
